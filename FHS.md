# file system hieraarchy
```
 bin: todos os binarios do sistema todos que podem ser executado por qualquer usuario.
 boot: arquivos de boot do sistema.
 dev: armazena todos os dispositivos do s.o.
 etc: diretorio de configuracao do sistema, qualquer programa que instale no s.o aqui tera os arquivos de configuracao.
 home: user system.
 lib: biblioteca do sistema.
 lib64: bibliotecas de 64byt.
 lost+found: achados e perdidos.
 mnt: ponto de montagem de midias, montagem temporaria(essa e' a diferenca dele pro /mount).
 opt: optional, pacotes separados que nao vem do pacote oficial.
 proc: sistema de arquivo, contem o pid de todos os processos e informacoes de hardher.
 root: super user system.
 run: aplicacoes em execucao quase como o /tmp, soq o run sao arquivos temporarios q estao em execucao por alguns programas, arquivos que precisam ficar em backgraud
 sbin: system bin binarios do s.o.
 srv: servir servisos.
 sys: semelhante ao /proc  informacoes de hieraarchy e drive do s.o
 tmp: execucao de arquivos temporarios. se arquivo tiver x tempo sem ser execultado ele e' deletado.
 usr: tipo o / soq apenas para os user
 var: variavel, tudo q nao pd ir pros outros diretorios vem pra esse.
```
